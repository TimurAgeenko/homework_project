# Проект "Виджет"

## Описание:

Проект "Виджет" - это виджет в приложении банка, который показывает несколько последних успешных банковских операций клиента.

## Установка:

1. Клонируйте репозиторий:
   ```
   git clone https://github.com/TimurAgeenko/homework_project
   ```
2. Установите зависимости:
   ```
   poetry install
   ```

## Использование:

1. Функцию get_mask_card_number можно использовать для создания замаскированного варианта номера карты:
   ```
   get_mask_card_number("7000792289606361")
   
   Выдаст "7000 79** **** 6361"
   ```
2. Функцию get_mask_account можно использовать для создания замаскированного варианта номера счета:
   ```
   get_mask_account("73654108430135874305")
   
   Выдаст "**4305"
   ```
3. Функция mask_account_card принимает тип и номер карты или счета и возвращает замаскированный вариант:
   ```
   mask_account_card("Visa Platinum 7000792289606361")
   
   Выдаст "Visa Platinum 7000 79** **** 6361"
   
   
   mask_account_card("Счет 73654108430135874305")
   
   Выдаст "Счет **4305"
   ```
4. Функция get_date принимает строку с датой в одном формате и возвращает в другом:
   ```
   get_date("2024-03-11T02:26:18.671407")
   
   Выдаст "11.03.2024"
   ```
5. Функция filter_by_state принимает список с информацией и значение state, по умолчанию равное 'EXECUTED' и возвращает отредактированный список
   со словарями, у которых ключ state принимает указанное значение:
   ```
   filter_by_state([
   {'id': 41428829, 'state': 'EXECUTED'},
   {'id': 939719570, 'state': 'EXECUTED'},
   {'id': 594226727, 'state': 'CANCELED'},
   {'id': 615064591, 'state': 'CANCELED'}
   ])
   
   Выдаст [
   {'id': 41428829, 'state': 'EXECUTED'},
   {'id': 939719570, 'state': 'EXECUTED'}
   ]
   ```
6. Функция sort_by_date принимает список и возвращает его отсортированный по дате вариант, по умолчанию - по убыванию:
   ```
   sort_by_date([
   {'id': 41428829, 'date': '2019-07-03T18:35:29.512364'},
   {'id': 939719570, 'date': '2018-06-30T02:08:58.425572'},
   {'id': 594226727, 'date': '2018-09-12T21:27:25.241689'},
   {'id': 615064591, 'date': '2018-10-14T08:21:33.419441'}
   ])
   
   Выдаст [
   {'id': 41428829, 'date': '2019-07-03T18:35:29.512364'},
   {'id': 615064591, 'date': '2018-10-14T08:21:33.419441'},
   {'id': 594226727, 'date': '2018-09-12T21:27:25.241689'},
   {'id': 939719570, 'date': '2018-06-30T02:08:58.425572'}
   ]
   ```
7. Функция filter_by_currency принимает список словарей с транзакциями и возвращает итератор,
   который поочередно выдает транзакции, где валюта операции соответствует заданной:
   ```
   generator_usd = filter_by_currency(transactions, "USD")
   next(generator_usd)
   
   Выдаст {
        "id": 939719570,
        "state": "EXECUTED",
        "date": "2018-06-30T02:08:58.425572",
        "operationAmount": {"amount": "9824.07", "currency": {"name": "USD", "code": "USD"}},
        "description": "Перевод организации",
        "from": "Счет 75106830613657916952",
        "to": "Счет 11776614605963066702",
    } 
   ```
8. Функция transaction_descriptions принимает список словарей с транзакциями
   и возвращает описание каждой операции по очереди:
   ```
   generator = transaction_descriptions(transactions)
   next(generator)
   next(generator)
   next(generator)
   
   Выдаст 
   "Перевод организации"
   "Перевод со счета на счет"
   "Перевод со счета на счет"
   ```
9. Функция card_number_generator принимает диапазон чисел
   и генерирует номера карт в указанном диапазоне, возвращая их по очереди
   ```
   generator = card_number_generator(1, 3)
   next(generator)
   next(generator)
   next(generator)
   
   Выдаст 
   "0000 0000 0000 0001"
   "0000 0000 0000 0002"
   "0000 0000 0000 0003"
   ```
10. Декоратор log используется для автоматической регистрации деталей выполнения функций,
    таких как имя функции, передаваемые аргументы, результат выполнения и информация об ошибках.
    Может записывать данные в файл, если указано имя файла, либо выводить их в консоль, если имя не указано.
   ```
   @log(filename=log.txt)
        def add_numbers(a, b):
            if not isinstance(a, int) or not isinstance(b. int):
                raise TypeError("Аргументы должны иметь тип int")
            return a + b
   
   result = add_numbers(1, 2)
   В файл log.txt добавится следующая строчка: add_numbers ok

   result = add_numbers("1", 2)
   В файл log.txt добавится следующая строчка:
   add_numbers error: Аргументы должны иметь тип int. Inputs: ('1', 2), {}

   Если же при обертывании функции в декоратор не будет указано название файла,
   то указанные строчки будут выведены в консоль 
   ```
11. Функция get_data выполняет десериализацию json файла и возвращает список словарей с данными о финансовых транзакциях.
    Если у данных неправильный формат, если использовать в функции путь к пустому файлу или к файлу,
    которого не существует, то она вернет пустой список.
   ```
   Для использования достаточно вызвать функцию
   с указанием пути до нужного файла:

   get_data("your_path_to_file")
   ```
12. Функция get_convert_amount принимает на вход сумму транзакции и её код: USD, EUR и т.д.,
    делает запрос для конвертации суммы в рубли и возвращает результат. 
    У суммы транзакции должен быть тип float, у кода - str.
   ```
   get_convert_amount(79114.93, "USD")
   
   Выдаст 6435830.40778
   ```
13. Функция get_transaction_amount принимает на вход словарь с данными о транзакции
    и возвращает сумму транзакции рублях, если в транзакции сумма указана не в рублях,
    то она использует функцию под номером 12, чтобы конвертировать сумму в рубли.
   ```
   get_transaction_amount({
       "id": 873106923,
       "state": "EXECUTED",
       "date": "2019-03-23T01:09:46.296404",
       "operationAmount": {"amount": "43318.34", "currency": {"name": "руб.", "code": "RUB"}},
       "description": "Перевод со счета на счет",
       "from": "Счет 44812258784861134719",
       "to": "Счет 74489636417521191160",
   })
   
   Выдаст 43318.34
   
   
   get_transaction_amount({
       "id": 873106923,
       "state": "EXECUTED",
       "date": "2019-03-23T01:09:46.296404",
       "operationAmount": {"amount": "79114.93", "currency": {"name": "USD", "code": "USD"}},
       "description": "Перевод со счета на счет",
       "from": "Счет 44812258784861134719",
       "to": "Счет 74489636417521191160",
   })
   
   Выдаст 6435830.40778
   ```
14. Функция get_csv принимает путь к файлу в формате csv, содержащий информацию о транзакциях,
    и возвращает список словарей с транзакциями.
   ```
   data = get_csv("./data/transactions.csv")
   data[0]

   'to': 'Счет 39745660563456619397',

   }
   ```
15. Функция get_excel принимает путь к файлу в формате xlsx, содержащий информацию о транзакциях,
    и возвращает список словарей с транзакциями.

## Тестирование

Для запуска тестов выполните следующие шаги:

1. Убедитесь, что у вас установлен `pytest`
2. В терминале наберите `pytest`, чтобы запустить тестирование
3. Проверьте результаты в консоли

При тестировании используются юнит-тесты, их результаты выводятся в консоль.

Подробные отчеты о тестировании можно найти в директории `/htmlcov/`
